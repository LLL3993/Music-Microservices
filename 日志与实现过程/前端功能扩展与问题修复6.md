# 前端美化日志（music-web）

## 目标与约束

- 目标：在不改变功能的前提下，美化 music-web 的整体观感，并让深色/浅色主题保持当前主色（红色 accent）一致与统一。
- 约束：不改动后端代码；不改动界面下方的音乐面板（底部 PlayerBar 的布局与样式选择器未调整）。
- 额外：在最底层增加缓慢流动、缓慢变化的色彩动效，作为整体背景氛围。

## 实现方式（设计思路）

- 采用“底层氛围 + 上层玻璃质感”的组合：
  - 最底层用多层径向渐变叠加，并用超长周期动画让渐变缓慢漂移。
  - 上层（顶部栏、侧边栏、弹窗遮罩）用半透明的面板色（glass）与 blur 形成统一的层次感，避免与底层动效冲突。
- 深色/浅色主题使用同一套语义变量：
  - 主题切换沿用现有 `data-theme="dark|light"` 机制，不引入新依赖。
  - 通过新增 `--panel-glass / --card-glass / --overlay / --aurora-*` 等变量，让两套主题在同一语义下得到不同的透明度与色彩强度。

## 关键变更

### 1) 底层缓慢流动背景（Aurora）

- 文件：`music-web/src/style.css`
- 变更点：
  - 为深色/浅色分别新增 `--aurora-1..4` 以及 `--overlay / --panel-glass / --card-glass`。
  - 使用 `#app::before` 与 `#app::after` 叠加多层径向渐变，并用 `aurora-drift` 做超长周期缓慢漂移。
  - 为了让“缓慢流动/缓慢变化”更可感知：提升 aurora 颜色透明度与整体饱和度，并将漂移动画调整为 `30s / 44s`，色相变化调整为 `120s / 180s`。
  - 增加 `prefers-reduced-motion: reduce` 的降级：系统偏好减少动画时自动关闭背景动效。

### 2) 布局层级与玻璃质感统一（不含底部音乐面板）

- 文件：`music-web/src/views/Layout.vue`
- 变更点：
  - `.layout` 从固定渐变背景改为透明，以便露出最底层 Aurora 背景。
  - `.topbar` 背景改为 `--panel-glass`，并使用响应式 padding（clamp）更适配不同分辨率。
  - `.sidebar` 改为 `--panel-glass` 并增加 blur，让侧边栏与顶部栏同一视觉语言。
  - `.content` 的背景改为带透明度的渐变，既能看到底层氛围，又保证阅读对比度。
  - `.btn.ghost` 去除硬编码浅色背景，改为基于 accent 与 card 的配色，保证深浅色一致性。
- 特别说明：
  - 未修改 `.playerbar` 以及 `.playerbar-*` 相关选择器，确保底部音乐面板不受影响。

### 3) 弹窗遮罩统一

- 文件：`music-web/src/components/OverlayMask.vue`
- 变更点：
  - 遮罩背景从固定 `rgba(0, 0, 0, 0.8)` 改为 `--overlay`，使浅色主题不再过暗，深色主题仍保持沉浸。
  - 弹窗内容背景改为 `--card-glass`，与整体玻璃质感一致。

- 文件：`seed-mysql.sh`


```bash

```
